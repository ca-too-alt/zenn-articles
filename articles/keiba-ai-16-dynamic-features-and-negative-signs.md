## 1. 概要

前回（第15話）では、バックテストにおける異常な高ROIの原因が「構造的データリーク」にあることを特定し、時点固定（Point-in-Time）特徴量生成による物理的なデータ隔離を実装しました。

リークを完全に排除したクリーンな環境で再評価を行った結果、穴馬検知モデルの的中率はランダムベースライン（約2.28%）と同等まで低下し、これまでの予測モデルが「未来の情報」に依存していたことが浮き彫りになりました。

第16話では、静的な過去集計データだけでは捉えきれない「穴馬の激走予兆」を識別するため、馬の状態変化や条件の好転を捉える「動的特徴量」の導入と、特定の条件下で的中率を著しく下げる「凡走サイン」の特定プロセスについて解説します。

## 2. 実装内容

単なる平均値や累積成績ではない、前走との比較や今回条件との適合性を測る指標群を新たに設計しました。主な実装内容は以下の通りです。

### 2-1. パフォーマンス変化の定量化
馬の成長や調子の変動を捉えるため、タイム偏差（パフォーマンススコア）を軸とした動的指標を導入しました。

- **`prev_performance_score`**: 前走のタイム偏差。
- **`perf_improvement`**: 「今回の想定能力」と「過去最高値」の差分。
- **`temp_position_score`**: 直近走での通過順位の変化。

### 2-2. 条件不適合のフラグ化（凡走サイン）
穴馬を当てることと同様に、過剰評価されている馬を除外することがROI向上には不可欠です。本実験では、特定の不適合条件を「凡走サイン」として定義しました。

### 2-3. 特徴量診断パイプラインの構築
新しい特徴量が真に予測に寄与しているかを検証するため、`01_diagnose_new_features.ipynb` を作成し、以下の指標で診断するフローを標準化しました。

- **KS統計量（Kolmogorov-Smirnov statistics）**: 激走馬と凡走馬の分布の乖離度。
- **Lift値**: 特徴量フラグが立っている場合の的中率が、ベースラインの何倍か。

## 3. 遭遇した問題

動的特徴量のプロトタイプ実装において、二つの大きな課題に直面しました。

### 3-1. 動的指標における論理的リークの再発
初期の `perf_improvement` の計算において、意図せず「今回のレース結果（タイム）」を計算式に含めてしまう論理的リークが発生しました。

```python
# リークしていた初期コード
df['perf_improvement'] = df['performance_score'] - df.groupby('horse_id')['performance_score'].transform('max')
```

このコードでは、当日の `performance_score` が過去最高を更新した場合、その「更新したという事実」が特徴量としてモデルに伝わってしまいます。これにより、見かけ上の精度が異常に高くなる現象が再発しました。

### 3-2. 決定木重要度と実性能の乖離
LightGBMの `feature_importance` では高い値を示すにもかかわらず、単体での的中率（Lift）が著しく低い特徴量が散見されました。代表的な例が、ダートから芝への転換を示す `is_surface_change` でした。これは「激走を予測する指標」としては機能しておらず、モデルが「この馬は走らない」という判断材料として強く利用している可能性が示唆されました。

## 4. 解決アプローチ

### 4-1. 「引き算」によるエッジの抽出
「激走サイン」を探すアプローチから、確実に的中率を下げる要因を排除する「引き算」のアプローチへ切り替えました。

具体的には、「ダート専門馬の芝挑戦」を捉える `is_dirt_specialist_on_turf` を導入。この特徴量は決定木で高い重要度を示しながらも、Lift値は **0.448**（ベースラインの半分以下）を記録しました。これを「凡走サイン」として定義し、後述する複合フラグのフィルタリングに利用しました。

### 4-2. 前走タイム偏差の再評価
診断の結果、`prev_performance_score`（前走のタイム偏差）がKS統計量 **0.1838** という高い分離能力を持つことが判明しました。これは、前走でハイレベルなレース（強い相手との接戦）を経験した穴馬が、今回相手弱化によって激走するパターンを統計的に捉えていることを意味します。

### 4-3. 複合フラグ `refined_anauma_flag_v2` の設計
単一の指標ではノイズが多いため、複数の条件を組み合わせた「高精度な激走シグナル」を構築しました。

1. 凡走サイン（`is_dirt_specialist_on_turf` 等）が立っていないこと
2. パフォーマンス改善（`perf_improvement`）が一定値以上であること
3. 前走着順が極端に悪すぎない（10着以内）こと

## 5. 最終的な解決策

物理的なデータリーク対策を施した `feature_generator.py` に、以下の動的ロジックを統合しました。

```python
# feature_generator.py 内での動的特徴量生成

# 1. 必ず shift(1) を行い、前走までの実績を基準にする
df = df.sort_values(['horse_id', 'start_datetime'])
df['prev_performance_score'] = df.groupby('horse_id')['performance_score'].shift(1)

# 2. 過去最高値の計算（これも shift(1) を介して今回を含めない）
df['historical_best_score'] = df.groupby('horse_id')['performance_score'].shift(1).cummax()

# 3. 自己ベスト更新度の算出
df['perf_improvement'] = df['prev_performance_score'] - df['historical_best_score']

# 4. 複合フラグの実装
df['refined_anauma_flag_v2'] = (
    (df['perf_improvement'] > 0) & 
    (df['is_dirt_specialist_on_turf'] == 0) &
    (df['prev_rank'] <= 10)
).astype(int)
```

この実装により、穴馬モデルのスコア（`score_anauma`）のKS統計量は、前回の **0.043** から **0.5087** へと劇的に改善しました。上位1%予測における的中率（Lift）は **1.359** を記録し、ランダムベースラインを有意に上回るエッジを確保することに成功しました。

## 6. 学んだこと

### 「凡走サイン」の特定は激走予測と同等に重要
決定木が重要視する特徴量には、「正の寄与（走る）」と「負の寄与（走らない）」の二種類があります。特に穴馬検知においては、期待値の低い「消し」の条件を正確に言語化し、フラグ化することで、モデルの探索空間が整理され、真の激走予兆を拾いやすくなることを学びました。

### 動的特徴量における論理的リークの執拗さ
`shift(1)` を一箇所忘れるだけで、モデルは即座に「答え」を学習してしまいます。新しく特徴量を追加した際には、必ずその特徴量単体でのKS統計量やLift値を算出し、非現実的な数値（例：Liftが10倍以上など）が出ていないかを確認する診断プロセスの重要性を再認識しました。

### 識別能の源泉は「変化」にある
馬の年齢や血統といった静的な属性だけでは、数万頭のデータから「今回だけ激走する1頭」を見つけ出すことは困難です。「前走からの上積み」や「不適合条件からの解放」といった動的な変化こそが、市場オッズを出し抜く（エッジを生む）主要な情報源であることを、統計的数値をもって証明できました。
