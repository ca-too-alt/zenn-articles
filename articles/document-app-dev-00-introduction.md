## はじめに
「プログラミングの経験はないけれど、自社に最適なシステムを自分たちの手で安く作りたい」

そんな無謀とも思える挑戦が、生成AIの進化によって現実のものとなりました。

本連載では、非エンジニアである私が、Gemini Code AssistやAmazon QといったAIツールの助けを借りながら、AWS Amplify Gen2を用いて実用的な「ドキュメント管理アプリ」を完成させるまでの軌跡を記録します。

## 課題：月額40万円という巨大なランニングコスト
これまで私の会社では、ドキュメント管理のために外部のグループウェアを使用していました。そのソフトはスケジュール管理やワークフローなど、非常に多機能なものでした。

しかし、私たちの主な用途は「書類の保管と版管理（バージョン管理）」です。使わない機能も多く含まれているにもかかわらず、そのランニングコストは月額約40万円。年間で500万円近い出費となっていました。

「必要な機能だけに絞り、自分たちで運用すれば、このコストはもっと抑えられるのではないか？」

これが、今回の開発プロジェクトの始まりでした。

## 狙い：サーバーレスアーキテクチャによる「持たない」開発
コストを最小限に抑えるため、私が注目したのはAWS（Amazon Web Services）の「サーバーレス」サービスです。

- **Amazon S3**: 堅牢で安価なファイルストレージ
- **Amazon DynamoDB**: 使った分だけ課金されるデータベース
- **AWS Lambda**: サーバーの管理が不要なプログラム実行環境

これらを組み合わせれば、サーバーを常時稼働させる必要がなくなり、利用料を劇的に抑えることができます。しかし、これらを一つずつ手動で設定し、プログラムを書くのは初心者には至難の業です。

そこで白羽の矢を立てたのが、AWSの最新開発フレームワーク **Amplify Gen2** でした。

## 開発スタイル：非エンジニア × 生成AIの二人三脚
私はエンジニアではありません。コードの書き方も、AWSの複雑な設定も、当初は全く分かりませんでした。

そこで、VS Code上で動作するAIアシスタント「Gemini Code Assist」や「Amazon Q Developer」をパートナーとして迎えました。

1. **日本語でやりたいことを伝える**: 「社員番号でログインできる仕組みを作りたい」「ファイル名が変わっても履歴を追えるようにしたい」とAIに相談します。
2. **AIがコードを生成する**: AIがTypeScriptによるインフラ定義やReactのコードを提案してくれます。
3. **エラーをAIと一緒に解く**: エラーが出たら、そのメッセージをそのままAIに貼り付けて原因と対策を一緒に考えます。

「できるところまでやってみよう」という軽い気持ちで始めたこのスタイルが、想像以上の力を発揮することになりました。

## 驚愕の結果：運用コストを97%削減
数ヶ月の試行錯誤を経て、アプリは無事に完成しました。そして、当初の目的であったコスト削減については、予想を遥かに上回る結果となりました。

- **旧システム（グループウェア）**: 月額 約400,000円
- **新システム（自作アプリ）**: ストレージ単体で月額 **約3,000円**
- **バックアップを2箇所に設定しても**: 月額 **約10,000円以内**

実に、**コストを97%以上削減**することに成功したのです。

## この連載でシェアしたいこと
このブログシリーズは、単なる「安く作れて良かった」という成功体験の羅列ではありません。

Amplify Gen2という新しい技術ゆえの「ハマりどころ」、非エンジニアだからこそ直面した技術的な壁、そしてAIへのプロンプト（指示出し）の工夫など、現場で起きたリアルな「悪戦苦闘の記録」です。

## 連載の構成（全10話）

本シリーズは以下のステップで、開発の裏側を公開していきます。

1. **プロジェクト始動と認証基盤の構築**: Amplify Gen2のセットアップ
2. **循環参照エラーとNode.jsの罠**: 開発初期に直面した最大の壁
3. **社員番号ログインの実装**: ビジネス要件に合わせたカスタム認証
4. **ユーザー管理システムの設計**: DynamoDBとCognitoの同期
5. **役割ベースのアクセス制御（RBAC）**: 権限管理の仕組み
6. **大規模リファクタリング**: メンテナンス性を高めるフォルダ構成
7. **堅牢なバージョン管理**: ファイル名変更に負けないID設計
8. **UXの追求**: 一括ダウンロードとプレビュー表示
9. **セキュリティと監視**: WAFとCloudWatchによる防御
10. **CI/CDと最新環境への移行**: Node.js 22への昇格と自動デプロイ

それでは、次回の第1話から、具体的な開発の様子を詳しくお届けします。
