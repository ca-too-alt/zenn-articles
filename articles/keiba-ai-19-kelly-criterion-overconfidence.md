## 1. 概要

前回の記事（第18話）では、モデル更新に伴うスコアの変動を吸収する「3フェーズ・ウォークフォワード基盤」を確立し、2026年Q1という未知の期間においても高い予測精度（スピアマン相関 0.40）を維持できることを確認しました。

「当てるための武器」が揃った今、次なる課題は「いかに効率よく稼ぐか」です。第19話では、投資理論において最強とされる資金管理術「ケリー基準（Kelly Criterion）」の導入と、それによって資産が数千円まで溶け落ちた、衝撃的な「数学的敗北」の記録を綴ります。

## 2. 実装内容：期待値に基づいた科学的投資

これまでの「一定金額を賭ける（ベタ買い）」から脱却し、予測確率とオッズから算出される「期待値（Expected Value: EV）」に基づいて投資額を動的に決定するロジックを `betting_logic.py` に実装しました。

### 2-1. ケリー基準の導入

ケリー基準とは、資産の成長率を最大化するための投資比率を算出する数式です。

$$f^* = \frac{p \times odds - 1}{odds - 1}$$

ここで、$p$ はモデルが予測した的中確率、$odds$ は確定オッズです。本プロジェクトでは、18頭立てのレースにおける極端なボラティリティを考慮し、算出した比率にさらに係数を掛ける「分数ケリー（Fractional Kelly）」を採用しました。

### 2-2. 期待値（EV）フィルタリング

予測確率と市場オッズを比較し、期待値が一定水準（`ev_threshold`）を超える買い目のみを抽出します。

```python
# betting_logic.py のロジック抜粋
def calculate_ev(prob, odds):
    return prob * odds

def calculate_kelly_bet(prob, odds, capital, multiplier=0.1):
    # ケリー基準による投資比率
    f_star = (prob * odds - 1) / (odds - 1)
    # 分数ケリー（安全のため multiplier を掛ける）
    bet_amount = capital * max(0, f_star) * multiplier
    return bet_amount
```

### 2-3. 月次ローリング最適化（Recalibration）

`SimulationEngine` を拡張し、1ヶ月ごとに「直近12ヶ月のデータで Optuna を回して戦略パラメータを最適化」→「翌月の実戦予測」を繰り返す、より実戦に近いウォークフォワード検証体制を構築しました。

## 3. 遭遇した問題：資産曲線の垂直落下

完璧な理論武装をして臨んだバックテスト（exp017）の結果は、目を疑うものでした。

*   **資産の推移**: 100万円でスタートした仮想資産が、2023年を境に急落。最終的に**数千円まで減少**し、実質的な「破産」を記録しました。
*   **異常な上振れと下振れ**: 一部のレースで資産の10%以上を突っ込み、外れるという挙動が頻発。
*   **ROIの乖離**: ベタ買いでのROIは 80% 超を維持しているにもかかわらず、資金管理を導入した途端に収支が壊滅しました。

なぜ、数学的に正しいはずのケリー基準が、資産を溶かす毒薬に変わったのでしょうか。

## 4. 解決アプローチ：信頼性曲線（Reliability Curve）による診断

原因を突き止めるため、モデルの予測確率 $p$ がどれほど「正しかったか」を検証する**信頼性曲線（Reliability Curve / Calibration Curve）**を分析しました。

### 4-1. 衝撃の事実：モデルの「過信」

分析の結果、現在のランキングモデルには致命的な数学的欠陥があることが判明しました。

*   **モデルの出力**: 「この馬が勝つ確率は **20%** です」
*   **実際の結果**: 100回試したところ、実際に勝ったのは **5回（5%）** だけ。

つまり、モデルが確率を **4〜5倍も過大評価（Over-confidence）** していたのです。

### 4-2. 数学的敗北のメカニズム

ケリー基準は「正確な的中確率」を前提とした式です。モデルが確率を5%から20%へ4倍過信すると、ケリー基準は「これはとてつもないお宝レースだ！」と判断し、資産の許容範囲を大幅に超える**オーバーベット**を指示します。

不正確な物差しで測った「期待値」に基づいて複利運用を行った結果、複利の力が資産を増やす方向ではなく、破滅させる方向にフル回転してしまったのが敗因でした。

## 5. 最終的な解決策：暫定的な「安全係数」の導入

根本的な解決には「確率の校正（Calibration）」が必要ですが、まずは現状のモデルで運用を継続するための応急処置として、**確率安全係数（`prob_safety_factor`）**を導入しました。

### 5-1. 的中確率の割引

モデルが算出した確率 $p$ を、Optuna で探索された安全係数（典型的に 0.2 前後）で一律に割り引く処理を追加しました。

```python
# 確率の過信を補正する
adjusted_prob = model_prob * trial.suggest_float('prob_safety_factor', 0.1, 0.5)

# 補正後の確率で EV とケリー比率を再計算
ev = calculate_ev(adjusted_prob, odds)
if ev > ev_threshold:
    bet_amount = calculate_kelly_bet(adjusted_prob, odds, current_capital)
```

### 5-2. 戦略パラメータの厳格な制約

Optuna の探索範囲を調整し、極端な投資を防ぐガードレールを設置しました。

*   `kelly_multiplier`: 0.01 〜 0.1（超クォーターケリー）
*   `ev_threshold`: 1.5 〜 2.5（より確実な歪みのみを狙う）

### 5-3. 改善後の資産推移

この補正により、資産の急激な崩壊は止まりました。2025年のシミュレーションでは **ROI 97.7%** まで回復し、JRAの払戻率（約75〜80%）を大きく上回る「エッジの種」を維持したまま、破産を回避できる戦略へと進化しました。

## 6. 学んだこと

### 「精度（Spearman相関）」と「確信（Calibration）」は別物である
これまでの実験（exp018まで）で追い求めていたのは、「馬の序列を正しく当てる能力（相関）」でした。しかし、馬券投資においては、「その確率がどれほど正確か」という**絶対的な校正精度**が、相関以上に重要であることを痛感しました。相関が 1.0 でも、確率が 2倍ズレていればケリー基準では破産します。

### 複利は諸刃の剣
期待値運用における複利は、予測が正確であれば資産を幾何級数的に増やしますが、予測に「過信」というノイズが混じった瞬間、資産を幾何級数的に破壊する暴力へと変わります。

## 7. 次回予告

モデルの「過信」を安全係数で無理やり抑え込むことで、破産は免れました。しかし、これは対症療法に過ぎません。私たちが次に成すべきは、数学的に正しい「物差し」を手に入れることです。

次回、モデルが吐き出す「嘘の確率」を「真の的中率」へと変換する、**確率校正（Calibration）**の本格的な実装に挑みます。
